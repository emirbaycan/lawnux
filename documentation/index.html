Lawnux Website Template

Lawnux - Law Firm & Lawyer Full Website Template

Lawnux is a Law Firm & Lawyer full website template built with HTML JS CSS PHP

It’s fully responsive and looks well. It is a modern design and the best speed performance template. This documentation will help to know more about template and how is structured and how to use the features.

Requirements

To be able to use HTML template, you must have the following:

  • Make sure you upload the required assets files/folders listed below:
    • css - Stylesheets Folder
    • js - Javacripts Folder
    • img - Images Folder
    • fonts - Fonts Folder
    • plugins - Plugins Folder
    • api - Php Folder
    • admin - Admin Panel Folder
    • writer - Author Panel Folder
    • home.html - Index File/Homepage

Installation

Follow the steps below to get started with your Lawnux Template:

  1. Open the Package Folder to find all the html Templates Files
  2. You will need to Upload all the files to localhost server or your Web Server using FTP in order to use it on your Website.
  3. In the Database Folder you need to execute the sql codes in your phpmyadmin to setup the database.
    • phpmyadmin - Create a database and connect a user with all privileges
    • /database/setup_data - Enter a username and password for your admin access to the management panel.
  4. In the Api Folder you need to copy all the files to your server in the same structure with the api
    • /api/lib/database - In this folder there is your connection to the database
    • $mysqli = new mysqli('localhost', 'username', 'password', 'database'); - Edit this line with your server configurations
  5. After you setup the server you can login from the /login url, and setup the pages
    • /messages - Messages that sent from your home and contact pages will come here, you can search, filter or delete them
    • /applications - Applications from the career page will come here with the cvs attached to them, you can also find the cvs in /cvs directory
    • /services - Services in the our-services page, home and in the footer will be displayed here, when you create a service its page automatically will be created with the header name urlized
    • /articles - Articles in the blog page will be displayed here, when you create an article its page automatically will be created with the header name urlized
    • /policies - Policies in the policies page will be displayed here, when you create an policy its page automatically will be created with the header name urlized
    • /users - You can create, edit or delete users from here
    • /writers - Writers in the writers page will be displayed here, when you create a writer its page automatically will be created with the header name urlized
  6. After you setup the pages with your data, you can edit all the texts in the html or php files and you can change all the images in the img folder

Api Structure

You should place your server infos in api/lib/database/Sql.php and mail infos in api/lib/mailerMailer.php

We splited the api files to general, lib, main, users folder to find the files easier
in general directory you will find the universal scripts,
in main directory the ones who can be called by visitors
in users the one who can be called by users

We splited them to add, get, update, upload directories and assosiciated the every file name by the html file name that it be in use

We are using api/lib/database/Sql.php module for every sql execution

We have api/lib/kalenux/Kalenux.php for creating the pages of article, service, policy

We have api/lib/file/File.php for creating, moving files in server

Change Fonts

To change the font, Check incss/general/general.css then first go to google fonts choose a font and use the generated code and replace with next code. 

                                        @font-face {
                                            font-family: "Montserrat";
                                            src: url(../../font/Montserrat/Montserrat-Regular.woff)format("woff");
                                            font-display: swap
                                        }

Next Step : change font family for html in the same place 

                                        html {
                                            background: var(--primary-color);
                                            font-family: "Montserrat";
                                            scroll-behavior: smooth
                                        }
